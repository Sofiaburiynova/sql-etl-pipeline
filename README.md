# SQL ETL Pipeline

Инкрементальный ETL-пайплайн для загрузки справочников в DWH (core layer) с обработкой изменений, дедупликацией и контролем качества данных.

---

## Описание проекта

Проект реализует загрузку данных из raw-слоя в core-слой хранилища данных с использованием SQL-процедур.

Основной фокус:
- инкрементальная загрузка данных
- отслеживание изменений через hash diff (md5)
- обработка новых, изменённых и удалённых записей
- обеспечение консистентности данных

---

## Реализованная логика

- Выделение бизнес-ключей (`_key`)
- Расчёт hash diff (`_md5`) для отслеживания изменений
- Дедупликация данных
- Инкрементальная загрузка:
  - `INSERT` — новые записи  
  - `UPDATE` — изменённые записи  
  - `SOFT DELETE` — удалённые записи  
- Использование буферного слоя (staging / buffer)
- Логирование этапов загрузки
- Обработка ошибок

---

## Структура проекта

```
sql/
 ├── 01_stg_operation_type.sql
 ├── 02_core_operation_type.sql
 └── 03_operation_type_validation.sql
```

---

## Архитектура

```
raw_data → staging (buffer) → core
```

- **raw_data** — источник данных  
- **staging / buffer** — промежуточный слой для обработки  
- **core** — целевой слой (очищенные и актуальные данные)  

---

## Технологии

- SQL (PostgreSQL / Greenplum)
- DWH подход (raw → core)
- ETL / ELT процессы

---

## Пример

ETL-процедура загрузки справочника `operation_type`:
- извлечение данных из `raw_data.payment_document`
- формирование бизнес-ключа
- сравнение через md5
- обновление и вставка записей
